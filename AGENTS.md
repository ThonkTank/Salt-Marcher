# SaltMarcher Agent Guide

Work on a feature branch, never commit to `main`, and run verification from the repo root. A pass without its required green command is WIP. Use `tools/gradle/run-staged-verification.sh production-handoff` for production code, `tools/gradle/run-staged-verification.sh focused-handoff --path <pkg> [--area <area>]` for package-limited checks, `./gradlew checkDocumentationEnforcement --console=plain` for docs/instruction Markdown, and `tools/gradle/run-staged-verification.sh desktop-install` after green production handoff when the desktop app is the manual surface.

Hard rules: do not weaken architecture/Error Prone checks to pass a build; mark retained transitional support with `LEGACY_REMOVE_ON_TOUCH`; record unresolved structural findings as `PROJECT_HEALTH_DEBT`; identify the owner before editing; behavior changes need an owning behavior harness or a `Harness Gap`; record L-tier design notes, incidents, repeated fixes, debt closures, and harness gaps in `docs/project/journal/YYYY-MM.md`.

Owners: `src/domain/**` -> `docs/project/architecture/patterns/domain-layer.md` + `domain-layer`; `src/features/**` -> `docs/project/architecture/patterns/feature-runtime.md` + `feature-runtime`; `src/view/**` -> `docs/project/architecture/patterns/view-layer.md` + `view-layer-mvvm`; `src/domain/dungeon/**` and `docs/dungeon/**` -> `docs/dungeon/README.md`; docs placement -> `docs/project/architecture/documentation.md`; verification -> `docs/project/verification/quality-platforms.md`; workflow/logs/instruction surfaces -> `docs/project/architecture/agent-instructions.md` plus `agent-instruction-engineering` for `AGENTS.md`, `SKILL.md`, and `agents/openai.yaml`; autonomous documentation-upkeep slices -> `docs/project/architecture/documentation.md` + `documentation-upkeep-steward`; external-source-backed decisions -> `docs/project/verification/source-references.md` + `source-references`.

Classify each tracked change by `docs/project/architecture/agent-instructions.md`: S edits directly with the matching gate; M records a short plan, proof, and judge review; L adds a journal design note before the M flow. Skills live in `tools/quality/skills/<name>/SKILL.md`; read a skill when this file, an owner doc, or the task surface names it.
