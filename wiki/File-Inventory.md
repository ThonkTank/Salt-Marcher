## Relevance Scale
Ratings reflect how strongly each file influences the user-facing features described across the wiki (Cartographer, Library, Encounter, UI, and supporting infrastructure).
- **10-9**: Core runtime assets required for the plugin to function in Obsidian.
- **8-7**: Build inputs, primary source files, and configuration that directly shape behaviour.
- **6-5**: Quality gates, generated bundles, and ancillary assets that keep features reliable.
- **4-3**: Documentation and supporting tooling with indirect impact.
- **2-1**: Peripheral artefacts (e.g. third-party docs) that do not materially affect features.

## Key Workflows
1. Consult the directory map above to understand how the repository is organised at a glance.
2. Use the comprehensive file listing below to locate specific assets, verify naming, or feed automation scripts. The current inventory is intentionally scoped to the `salt-marcher/` plugin package.
3. Refresh the inventory after structural changes by regenerating the list with `find ./salt-marcher -type f -print | sort` from the repository root.
