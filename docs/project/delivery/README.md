# Project Delivery

Delivery documents own temporary execution state, never stable architecture or
feature behavior. Each contract names its goal, adopted target, milestones,
current position, writer allocation, blockers, next transition, neighboring
canonical owners, and deletion condition.

- [Verification Harness Replacement](verification-harness-replacement/README.md)

Delete a delivery contract and its index entry after its completion facts are
recorded in stable owners or Git history.
