# July 2026 Journal

## 2026-07-04 governance-simplification - Simplify agent workflow governance

The governance model at baseline commit `ef8c7eb6d` used a CR, roadmap,
phase/wave-plan, plan-review, work-log, and coordinator workflow.
This migration replaces that ceremony with S/M/L tiers: S edits directly with a
matching gate, M records a short plan plus proof and judge review, and L adds a
one-page journal design note before the M flow.
Generated pass logs remain transient under `build/agent-pass-logs/`; durable
decisions, incidents, repeated fixes, debt closures, and harness gaps are
recorded here.

## 2026-07-04 documentation-upkeep-steward - Route autonomous documentation upkeep

Problem: the autonomous documentation-upkeep design needs a repo-local route so
the steward can improve navigation without becoming an unbounded truth editor.
Target state: `AGENTS.md` points autonomous documentation-upkeep slices to a
small `documentation-upkeep-steward` skill, while ordinary documentation edits
keep their normal owner routing.
Alternatives rejected: changing `documentation.md` now would mix the requested
repo hook with token-gated model evolution; implementing Warden checks in this
repo would create a second owner for external automation.
Scope: add the steward skill and one routing row only; no Warden, Gradle,
production-code, truth-doc, or source-mirror changes.
Done when: documentation enforcement passes and review confirms the skill is a
router that preserves S1-S5 safety boundaries.

## 2026-07-05 target-operating-model - Implement autonomous delivery gates

Problem: SaltMarcher needs autonomous technical delivery without owner-side
technical decisions, red-CI normalization, or behavior changes that bypass
acceptance.
Target state: `main` is protected by production-handoff, warden-freeze,
behavior-gate, and judge-review; stable releases come from tags; owner-facing
German issue, status, and acceptance surfaces hold product decisions.
Alternatives considered: keeping SonarCloud and CodeScene as required gates was
rejected because third-party uptime should not block 24/7 autonomy; a repo-local
agent runner was rejected because it would add tokens, scheduler state, and
abuse surface beyond this repository's gate responsibilities.
Scope boundary: implement repo-owned gates, scripts, docs, and local updater
surfaces; owner actions remain labels, secrets, branch protection, policy
signoff, template render check, and updater install.
Done when: documentation and production handoff gates pass locally, live
branch-protection readback later reports `Qualified`, and owner actions are
ticked in the status report.
