## 2026-07-04 governance-simplification - Simplify agent workflow governance

The governance model at baseline commit `ef8c7eb6d` used a CR, roadmap,
phase/wave-plan, plan-review, work-log, and coordinator workflow.
This migration replaces that ceremony with S/M/L tiers: S edits directly with a
matching gate, M records a short plan plus proof and judge review, and L adds a
one-page journal design note before the M flow.
Generated pass logs remain transient under `build/agent-pass-logs/`; durable
decisions, incidents, repeated fixes, debt closures, and harness gaps are
recorded here.

Done when: documentation enforcement passes and review confirms the skill is a

## 2026-07-05 phase2-activation-judge-oauth - Use Claude Code OAuth token for judge

The OAuth token successfully called `/v1/messages` with `claude-opus-4-8`
when using the Claude Code system prompt and `oauth-2025-04-20` beta header.

Claude Code CLI and falls back to `ANTHROPIC_API_KEY` when no OAuth token is

`/v1/messages` bearer token, so the OAuth path must run through Claude Code
itself. `ANTHROPIC_AUTH_TOKEN` is only a temporary secret alias mapped into
`CLAUDE_CODE_OAUTH_TOKEN` until the canonical secret name is available.
